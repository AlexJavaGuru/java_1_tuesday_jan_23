Типизированные — это те, которые различают разные типы данных, например: Java, JavaScript, PHP, C, C++, C#, Python
и др.;
Типизированные языки деляться на статические/динамические, слабые/сильные, явная/неявная
Со статической типизацией проверка типов начинается на стадии компиляции. Эти языки зачастую быстрее. Тип для
пересенной можно назначить только один раз.
С динамической типизацией ищют ошибки на стадии исполнения. В них можно задавать разные типы переменных для одной и
той же и они более гибкие.

‌выполнять‌ ‌операции‌ ‌преобразования‌ ‌с‌ ‌разными‌ ‌типами. В ‌таком‌ ‌случае‌ ‌не‌ ‌всегда‌ ‌ясно, как‌ ‌поведёт‌ ‌себя‌
‌программа.
При‌ ‌сильной‌ ‌типизации, ‌язык‌ ‌не‌ ‌позволяет‌ ‌смешивать‌ ‌разные‌ ‌типы‌ ‌—‌ ‌то‌ ‌есть, ‌если‌ ‌вы‌ ‌обозначили‌ ‌переменную‌ ‌как‌ ‌число,
‌то‌ ‌добавить‌ ‌к‌ ‌ней‌ ‌строку‌ ‌уже‌ ‌не‌ ‌получится. Сильная типизация надежнее.
С‌ ‌явной‌ ‌типизацией‌ ‌типы‌ ‌переменных‌ ‌и‌ ‌возвращаемых‌ ‌значений‌ ‌функций‌ ‌нужно‌ ‌задавать.

Нетипизированные — это те, которые не различают никаких типов данных, например: язык ассемблера, Forth, Brainfuck.

Java является типизированным языком. Статическая, сильная и явная типизация.
